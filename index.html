拒止环境中的协同作战

发表于 2021-06-28 分类于 F 无人机集群系统 ， F1 无人机集群概述 ， F1-1 无人机集群项目

项目名称：Collaborative Operations in Denied Environment (CODE)
主持单位：Tactical Technology Office (TTO), Defense Advanced Research Projects Agency (DARPA)
开始时间：2015年
项目简介：Modular software architecture for UAVs to pass information to each other in contested environments to identify and engage targets with limited operator direction.

项目简介

所谓拒止环境，是美军对强对抗环境的别称，在这种环境中，强大对手使用军事力量对抗美军进入战区实现作战目的。
2014年（存疑？），美国国防部高级研究计划局（Defense Advanced Research Projects Agency, DARPA）战术技术办公室（Tactical Technology Office, TTO）启动“拒止环境中的协同作战”（Collaborative Operations in Denied Environment, CODE）项目，旨在研发先进的自主协同算法和监督控制技术以增强无人机在拒止环境中的协同作战能力，这里无人机包括巡航导弹、诱饵无人机及其他无人机系统。关键技术涉及四个方面：编队协同自主、飞行器自主、监控界面，以及适用于分布式系统的开放架构。关键技术的发展注重自主协同作战传感、打击、通信和导航、低通信带宽、人机交互等方面。

李磊,王彤,蒋琪.美国CODE项目推进分布式协同作战发展[J].无人系统技术,2018,1(03):59-66.

项目进展

  1. 2015年1月，DARPA发布了一份邀请函，邀请业界参与CODE的第一阶段工作，并指出了无人机系统的优点，但认为它们在远距离作战和拒止环境中的能力有限，因为它们需要持续的、专门的控制。详见Flight Global网站。
  2. 2016年6月，DARPA将第二阶段的CODE系统集成合同授予了洛马公司和雷神公司。在CODE项目的第二阶段，DARPA计划在两个开放式架构团队内均实施一种初始行为子集，并使用那些架构通过一个或两个真实UAS协同几个虚拟飞机执行真实的飞行测试。如果测试成功，DARPA将开展第三阶段研究，即其中一个团队将测试六个UAS并协同其他模拟飞行器合作的能力。试验将通过单人指挥UAS团队执行复杂的任务，包括目标搜索、识别以及对主动的、不可预知的对手的战斗。详见UAS Vision网站。
  3. 2018年1月，DARPA选择雷神公司在项目第三阶段完成CODE软件的研发，详见国防科技信息网，UAS Vision网站。
  4. 2018年11月，在亚利桑那州尤马试验场最近的一系列测试中，DARPA的CODE项目展示了装备CODE的无人机系统（UASs）在反进入拒止环境中适应和应对意外威胁的能力。
  5. 2019年3月，DARPA宣布：近日在亚利桑那州尤马试验场进行CODE项目中，一群无人机在通讯中断、GPS系统无法使用的情况下，成功完成了预定的任务目标。此次系列试验首先由地面小型作战中心的任务团队创建了一个真实/虚拟结合的测试环境，负责跟踪由6架RQ-23“虎鲨”无人机组成的无人机群，并在航空地图上虚拟了多达14架飞机。在4次演示飞行期间，任务团队激活了各种虚拟目标、威胁和对抗措施，以检验RQ-23无人机在不理想的环境下完成任务目标的能力。详见国防科技信息网，UAV vision网站。
  6. 2020年12月，美国通用原子公司表示，在没有飞行员参与的情况下，DARPA的CODE软件控制“复仇者”无人机机动工作了两个多小时。实验性CODE程序旨在协助美军使一架或多架无人机自主飞行，而无需始终保持通信链路连接，也不需要专门的飞行员和传感器操作员。理论上，自主飞行将使更多的无人机同时飞行，并能够防止无人机的输入控制信号被干扰或欺骗。此次飞行试验的成果有助于推进“空中博格”（Skyborg）项目，该项目由美国空军研究实验室主导，旨在为各种无人机开发自主飞行和任务控制程序。详见国防科技信息网。

个人博客管理

发表于 2021-06-28 分类于 Z 综合类 ， Z1 资料管理 ， Z1-1 在线笔记管理

1. 准备工作

1.1 Github

  • 注册Gitbub账号
  • 新建Repository，名称为你的Github账号名.github.io，如PlutoKing.github.io

  • 前往博客查看，你的github用户名.github.io，如plutoking.github.io。

  这里是自动生成的README.md的内容.

1.2 安装Git

  • Git网站，点击下载

  • 安装

1.3 安装Node.js

  • Node.js官网

  • 点击12.14.0 LTS下载（左侧为最新版本）
  • 安装
  • 打开Node.js command prompt
  • 输入node -v查看版本，可以看到v12.14.0
  • 输入git --version查看git版本，可以看到git version 2.24.1.windows.2

1.4 安装Hexo

  • 依然在Node.js command prompt中，输入npm install -g hexo-cli安装Hexo
  • 输入hexo -v查看Hexo版本，可以看到hexo-cli: 3.1.0等多行参数

2. 搭建个人博客

2.1 创建本地博客

  • 选择本地博客文件夹，如F:\Documents，在其中新建文件夹GitBlog
  • 在新建的文件夹中，右击，选择Git Bash Here，打开Git Bash
  • 输入hexo init将GitBlog文件夹初始化成一个博客文件夹
  • 输入npm install安装依赖包
  • 输入hexo g生成（generate)网页

    由于还没有创建任何博客，生成的网页会展示Hexo自带的一个Hello World的博客。

  • 输入hexo s将生成的网页放在了本地服务器（server）
  • 浏览器中输入http://localhost:4000/，可以查看本地服务器中的博客首页

  • 回到Git Bash，按 Ctrl+C 结束

    此时再看http://localhost:4000/就是无法访问了。

2.2 写一篇博客

  • 依然在Git Bash中，输入hexo n "New Post"，New Post为新建博客名称
  • 在F:\Documents\GitBlog\source_posts文件夹下，会有一个New Post.md文件，使用markdown编辑和这个文件，然后保存 ==（如何编辑markdown文件，详见本文第4小节）==
  • 回到Git Bash，输入hexo g，生成文件，可以在public文件夹中查看
  • 输入hexo s
  • 前往http://localhost:4000/查看结果
  • 回到Git Bash，按 Ctrl+C 结束

2.3 部署博客

  • 在本地博客文件夹中，打开_config.yml，这是==站点配置文件==
  • 修改基本配置
    • title后面输入博客的标题
    • author后面输入作者
    • language后面可以修改语言，这里改为中文
  • 修改部署配置
    • type后添加git，表示发布方式
    • 添加repository字段，并添加博客在网上存储的仓库的网址
    • 添加branch字段，值为master
  • 回到Git Bash，输入npm install hexo-deployer-git --save，安装hexo的git部署工具
  • 输入hexo d，将本地博客部署到网上
  • 得到NFO Deploy done: git，部署成功
  • 前往PlutoKing.github.io查看结果

至此，个人博客已经搭建完成

3. Hexo主题配置

Hexo的主题可以查看其官网Hexo主题。这里以NexT主题为例。（NexT的Demo）

4. VS Code配置Markdown编辑器

个人博客已经搭建完毕，那么如何编辑博客就是下一个问题。hexo n "title"生成的title.md是一个Markdown文件，所以需要一个Markdown的编辑器。我们选择VS Code作为Markdown编辑器。

4.1 下载安装VS Code

  • 前往VS Code官网
  • 下载VS Code
  • 安装

4.2 添加VS Code插件

  • 点击右侧工具栏第五个图标：插件

  • 搜索chinese，安装中文包
  • 搜索Markdown Preview Enhanced，安装Markdown的预览增强插件

4.3 使用VS Code写博客

  • 打开文件夹，选择source/_post文件夹
  • 新建一个md文件，如个人博客管理.md

  • 文件头说明
    • title 后面是博客标题
    • date 后面是日期，是博客归档的依据
    • tag 后面是标签，如本文标签为Blog
    • categories后面为分类，本文采用了二级分类（如果有多个标签，也采用分类的这种写法。

  • 按照Markdown的语法撰写博客内容
  • 按Ctrl+K V，打开Markdown预览界面

至此，就可以快乐地使用VS Code编辑博客了；编辑完成后，在本地博客文件夹中右击，选择Git Bash here，输入hexo g -d，便可以生成博客文件，并部署到Github；再打开plutoking.github.io便可以查看结果。
